# UI layout priority pass v16

## What changed
- Purchases now opens with the purchase composer before overview metrics, because creation is the primary daily action.
- Reports now shows section tabs immediately under the header, so the user picks the report first before reading cards or metrics.
- Settings now shows section tabs before the guidance strip in normal mode, so the user picks the correct settings area first.
- Audit now opens with workflow guidance and the searchable activity register before metrics, so review starts from the log itself.

## Intent
Prioritize the first decision on each page, then show supporting context, then secondary metrics/actions.
