# UI layout optimization pass

هدف المرحلة:
- جعل أول ما يراه المستخدم داخل كل شاشة هو الجزء الأكثر استعمالًا فعليًا.
- تقديم السجل/البحث قبل النماذج في الشاشات المرجعية.
- تثبيت شريط أولوية سريع أعلى الشاشات التشغيلية.

الشاشات المعدلة:
- dashboard
- pos
- products
- customers
- suppliers
- services
- cash-drawer

أهم القرارات:
- الكاشير يركز أولًا على البحث + السلة + الإتمام.
- الأصناف/العملاء/الموردون/الخدمات تعرض السجل أولًا ثم التحرير/الإضافة.
- الرئيسية والورديات تعرض شريط أولوية واضح يحدد ما يجب مراجعته أولًا.
- بقية الشاشات التي كانت تحتوي أصلًا على guidance/spotlight أو ترتيب صحيح لم تُعد هيكلتها بدون داعٍ.
